# BUILD.md — Technical Build Process

Internal document describing the FocusLock build pipeline, PyInstaller configuration,
and distribution packaging. Not intended for end users.

---

## 1. Overview

The build produces a self-contained Windows distribution from the Python source tree.
The target user has **no Python installation**. Everything — the interpreter, all
dependencies, and all app modules — is bundled into a single folder via PyInstaller's
`--onedir` mode.

```
Source tree (src/)  →  PyInstaller  →  dist/FocusLock-3.0.0/
                                         ├── FocusLock.exe       (bootloader + PKG)
                                         ├── _internal/           (runtime + deps)
                                         ├── Install.bat
                                         ├── Uninstall.bat
                                         └── README.txt
```

---

## 2. Entry Point

```
src/focuslock_app.py
```

This is the single file PyInstaller analyzes. All other modules are resolved through
Python imports at analysis time. The `--paths src` flag adds `src/` to `sys.path` so
that `from focuslock.xxx import yyy` resolves correctly.

## 6. Build Commands

### Development build (for testing)

```bash
python build.py --fast --dist
```

- Skips clean step
- Builds to `dist/FocusLock/`
- Creates distribution folder at `dist/FocusLock-3.0.0/`
- Typical build time: ~35 seconds

### Full build (clean + build + dist)

```bash
python build.py --dist
```

- Removes `build/`, `dist/`, all `__pycache__/`
- Full rebuild from scratch
- Creates distribution folder

### Clean only

```bash
python build.py --clean
```

### Via batch file (Windows)

```bash
build.bat              # Full build
build.bat --fast       # Fast build
build.bat --clean      # Clean only
```

## 7. build.py Internals

### Step 1: check_prerequisites()
- Verifies Python >= 3.10
- Checks for PySide6, SQLAlchemy, psutil, pyinstaller
- Auto-installs missing packages via pip

### Step 2: clean()
- Removes `build/`, `dist/`, `build/build_tmp/`
- Walks `src/` removing all `__pycache__/` directories

### Step 3: build()
Constructs the PyInstaller command:

```
pyinstaller
  --noconfirm
  --clean
  --onedir                     # Folder-based output (not --onefile)
  --windowed                   # No console window
  --name FocusLock
  --paths src                  # Adds src/ to module search path
  --hidden-import <30+ modules>
  --exclude-module <6 modules>
  --distpath dist
  --workpath build/build_tmp
  --specpath .
  src/focuslock_app.py
```

The `--onedir` flag is critical: it produces a folder containing `FocusLock.exe`
plus `_internal/` with all runtime files. This is more reliable than `--onefile`
(which extracts to a temp directory at runtime and can trigger antivirus false positives).

### Step 4: create_distribution()
Copies `dist/FocusLock/` to `dist/FocusLock-3.0.0/` and adds:
- `Install.bat` — creates a Desktop shortcut via PowerShell
- `Uninstall.bat` — removes the Desktop shortcut
- `README.txt` — end-user instructions

## 8. FocusLock.spec

Auto-generated by PyInstaller during the build. The spec file records the exact
analysis, PYZ, EXE, and COLLECT configuration used. Key settings:

```python
a = Analysis(
    ['src/focuslock_app.py'],
    pathex=['src'],
    hiddenimports=[...],     # 30+ modules
    excludes=[...],          # 6 excluded modules
)
pyz = PYZ(a.pure)
exe = EXE(pyz, ..., console=False, name='FocusLock')
coll = COLLECT(exe, a.binaries, a.datas, name='FocusLock')
```

The spec is regenerated on each build (`--clean` flag in PyInstaller). Manual edits
to the spec file are overwritten. All configuration is driven by `build.py` command-line
arguments.

## 9. GitHub Actions CI/CD

**File:** `workflows/build.yml`

**Trigger:** Push tag `v*` or manual dispatch.

### Job: build-windows
1. Checkout code
2. Setup Python 3.11 with pip cache (`actions/setup-python@v5`)
3. `pip install -r requirements.txt && pip install pyinstaller`
4. `python build.py --dist`
5. Upload `dist/FocusLock-*/` as artifact (full distribution)
6. Upload `dist/FocusLock/FocusLock.exe` as artifact (exe only)
7. On tag push: create GitHub Release with the exe (`softprops/action-gh-release@v2`)

### Artifacts produced
- `FocusLock-v<tag>-Windows` — full distribution folder
- `FocusLock-v<tag>-exe` — standalone exe

## 11. Runtime Requirements

The built distribution requires:
- Windows 10 or later (x64)
- No Python installation
- No additional DLLs or runtimes (VCRUNTIME140.dll is bundled)
- Administrator privileges only for website blocking (hosts file modification)

---

## 12. Database Location

At runtime the app creates:
```
%APPDATA%/FocusLock/focuslock.db    (SQLite, WAL mode)
```

This is outside the build output. Uninstalling the app does not remove user data.
To fully reset: delete `%APPDATA%/FocusLock/`.

## 13. Adding an Icon

1. Place a `.ico` file at `assets/icon.ico`
2. Rebuild: `python build.py --dist`
3. The `--icon assets/icon.ico` flag is added automatically when the file exists

## 14. Troubleshooting

| Issue | Fix |
|-------|-----|
| `ModuleNotFoundError` at runtime | Add missing module to `--hidden-import` list in `build.py` |
| Antivirus flags the exe | Use `--onedir` (already default). `--onefile` extracts to temp and triggers more false positives |
| Build fails with ` RecursionError` | Increase recursion limit or add `--exclude-module` for unused heavy packages |
| `_internal/` missing DLLs | Check `--hidden-import` for the affected package; PyInstaller hooks may not cover all submodules |
| Website blocker doesn't work | App must be run as Administrator (hosts file requires elevation) |
